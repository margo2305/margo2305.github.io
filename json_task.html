JSON и сериализация в C++

Анализ доклада: «Неожиданные подводные камни»

Обоснование выбора темы

Выбор данной темы обусловлен высокой актуальностью эффективной обработки данных в современной разработке. JSON является стандартом обмена данными, а С++ — языком, где производительность стоит на первом месте.

Мне было интересно изучить, как решаются классические проблемы «велосипедостроения»: почему стандартных средств языка иногда недостаточно, как избежать лишних аллокаций памяти и зачем нужна ручная валидация UTF-8. Доклад Павла Новикова ценен тем, что он показывает путь от простой идеи до оптимизированной библиотеки min_json_cpp, разбирая ошибки, которые совершают даже опытные разработчики.

Практический пример: min_json_cpp

Пример создания объекта с данными о книге «JSON in C++» и его сериализации:


#include "min_json.hpp"
#include <iostream>

int main() {
    min_json::object_t book_info = {
        {"title", "JSON in C++"},
        {"author", "Pavel Novikov"},
        {"pages", 1500},
        {"topics", min_json::array_t{"Serialization", "UTF-8", "Optimization"}},
        {"is_active", true}
    };

    std::string output = min_json::serialize(book_info, min_json::options::pretty_print);
    std::cout << output << std::endl;

    return 0;
}
